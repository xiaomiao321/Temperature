# 74HC595 驱动数码管移植指南

## 一、74HC595 芯片简介

### 1.1 引脚说明
| 引脚 | 名称 | 功能 |
|------|------|------|
| 14 | DS (SER) | 串行数据输入 |
| 11 | SHCP (SRCLK) | 移位寄存器时钟 |
| 12 | STCP (RCLK) | 输出寄存器时钟 (锁存) |
| 10 | /MR | 主复位 (低电平有效，通常接 VCC) |
| 9 | /Q7' | 串行数据输出 (用于级联) |
| 15, 1-7 | Q0-Q7 | 并行数据输出 |
| 8 | GND | 地 |
| 16 | VCC | 电源 |

### 1.2 工作原理
```
┌─────────────────────────────────────────────────────────┐
│  串行输入 → [8 位移位寄存器] → [8 位锁存器] → 并行输出  │
│      ↑              ↑               ↑                   │
│     DS           SHCP            STCP                  │
│   (数据)        (移位时钟)      (锁存时钟)              │
└─────────────────────────────────────────────────────────┘

工作时序:
1. DS 引脚送入数据位
2. SHCP 上升沿 → 数据移入移位寄存器
3. 重复 8 次，完成 1 字节传输
4. STCP 上升沿 → 数据锁存到输出引脚
```

## 二、STM32CubeMX 引脚配置

### 2.1 最少引脚需求 (单片 74HC595)
| 功能 | 信号名 | CubeMX 配置 | 说明 |
|------|--------|-------------|------|
| 数据 | DS | GPIO_Output | 推挽输出 |
| 移位时钟 | SHCP | GPIO_Output | 推挽输出 |
| 锁存时钟 | STCP | GPIO_Output | 推挽输出 |

### 2.2 CubeMX 配置步骤

#### 步骤 1: 选择任意 3 个 GPIO 引脚

推荐使用同一端口的连续引脚，例如：
- **方案 A**: PA5, PA6, PA7 (推荐，便于软件 SPI 移植)
- **方案 B**: PB5, PB6, PB7
- **方案 C**: 任意可用 GPIO

#### 步骤 2: 配置引脚模式

在 CubeMX Pinout 视图中：
1. 点击选中的引脚
2. 设置为 **GPIO_Output**
3. 在左侧 **System Core → GPIO** 中配置：

| 参数 | 推荐值 |
|------|--------|
| GPIO mode | Output Push-Pull (推挽输出) |
| GPIO Pull-up/Pull-down | No pull-up and no pull-down |
| Maximum output speed | High (50MHz 或更高) |
| User label | DS/SHCP/STCP (便于识别) |

#### 步骤 3: 时钟配置
确保系统时钟 ≥ 8MHz，推荐 72MHz (F103) 或更高。

#### 步骤 4: 生成代码
- Project Manager → 选择 IDE (MDK-ARM / IAR / Makefile)
- 点击 **Generate Code**

## 三、代码实现

### 3.1 添加驱动文件
在 `Core/Src` 和 `Core/Inc` 目录下创建文件：

### 3.2 头文件 `hc595.h`
```c
#ifndef __HC595_H
#define __HC595_H

#include "main.h"
#include <stdint.h>

/* ========== 用户配置区 ========== */
/* 根据 CubeMX 实际引脚修改 */
#define HC595_PORT          GPIOA
#define HC595_DS_PIN        GPIO_PIN_5      // 数据引脚
#define HC595_SHCP_PIN      GPIO_PIN_6      // 移位时钟
#define HC595_STCP_PIN      GPIO_PIN_7      // 锁存时钟
/* ================================ */

/* 共阳极数码管段码表 0-9, A-F, 灭, 点 */
extern const uint8_t SEG_CODE[];

/* 函数声明 */
void HC595_Init(void);
void HC595_SendByte(uint8_t data);
void HC595_Latch(void);
void HC595_Display(uint8_t num);
void HC595_DisplayRaw(uint8_t seg_code);

/* 级联支持 */
void HC595_SendMultiBytes(uint8_t *data, uint8_t count);

#endif /* __HC595_H */
```

### 3.3 源文件 `hc595.c`
```c
#include "hc595.h"
#include "main.h"

/* 共阳极段码表: 0-9, A-F, 灭，点 */
const uint8_t SEG_CODE[] = {
    0xC0,  /* 0 */
    0xCF,  /* 1 */
    0xA4,  /* 2 */
    0xB0,  /* 3 */
    0x99,  /* 4 */
    0x92,  /* 5 */
    0x82,  /* 6 */
    0xF8,  /* 7 */
    0x80,  /* 8 */
    0x90,  /* 9 */
    0x88,  /* A */
    0x83,  /* B */
    0xC6,  /* C */
    0xA1,  /* D */
    0x86,  /* E */
    0x8E,  /* F */
    0xFF,  /* 灭 */
    0x7F   /* 点 */
};

/**
 * @brief  短延时 (用于 74HC595 时序)
 *         CubeMX 生成的 HAL 库版本，根据实际时钟调整
 */
static void HC595_Delay(void)
{
    __NOP();
    __NOP();
    __NOP();
}

/**
 * @brief  初始化 74HC595 引脚
 *         如果在 CubeMX 中已配置为输出，此函数可省略
 */
void HC595_Init(void)
{
    /* CubeMX 已初始化 GPIO，此处可选 */
    HAL_GPIO_WritePin(HC595_PORT, HC595_DS_PIN | HC595_SHCP_PIN | HC595_STCP_PIN, GPIO_PIN_RESET);
}

/**
 * @brief  发送 1 字节数据到 74HC595
 * @param  data: 要发送的数据
 */
void HC595_SendByte(uint8_t data)
{
    uint8_t i;
    
    for(i = 0; i < 8; i++)
    {
        /* 1. 拉低移位时钟 */
        HAL_GPIO_WritePin(HC595_PORT, HC595_SHCP_PIN, GPIO_PIN_RESET);
        
        /* 2. 设置数据位 (MSB First) */
        if(data & 0x80)
            HAL_GPIO_WritePin(HC595_PORT, HC595_DS_PIN, GPIO_PIN_SET);
        else
            HAL_GPIO_WritePin(HC595_PORT, HC595_DS_PIN, GPIO_PIN_RESET);
        
        /* 3. 数据移位 */
        data <<= 1;
        
        /* 4. 短延时 */
        HC595_Delay();
        
        /* 5. 拉高移位时钟 (上升沿移位) */
        HAL_GPIO_WritePin(HC595_PORT, HC595_SHCP_PIN, GPIO_PIN_SET);
        
        /* 6. 短延时 */
        HC595_Delay();
    }
    
    /* 确保最终状态 */
    HAL_GPIO_WritePin(HC595_PORT, HC595_SHCP_PIN, GPIO_PIN_RESET);
}

/**
 * @brief  锁存数据到输出
 */
void HC595_Latch(void)
{
    HAL_GPIO_WritePin(HC595_PORT, HC595_STCP_PIN, GPIO_PIN_SET);
    HC595_Delay();
    HAL_GPIO_WritePin(HC595_PORT, HC595_STCP_PIN, GPIO_PIN_RESET);
}

/**
 * @brief  显示单个数字 (0-15)
 * @param  num: 要显示的数字
 */
void HC595_Display(uint8_t num)
{
    if(num > 15) num = 15;
    HC595_SendByte(SEG_CODE[num]);
    HC595_Latch();
}

/**
 * @brief  显示原始段码
 * @param  seg_code: 段码值
 */
void HC595_DisplayRaw(uint8_t seg_code)
{
    HC595_SendByte(seg_code);
    HC595_Latch();
}

/**
 * @brief  发送多字节 (用于级联)
 * @param  data: 数据缓冲区
 * @param  count: 字节数
 */
void HC595_SendMultiBytes(uint8_t *data, uint8_t count)
{
    uint8_t i;
    for(i = 0; i < count; i++)
    {
        HC595_SendByte(data[i]);
    }
    HC595_Latch();
}
```

## 四、使用示例

### 4.1 单个数码管显示
```c
/* main.c */
#include "hc595.h"

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    
    HC595_Init();
    
    while(1)
    {
        /* 显示数字 0-9 循环 */
        for(uint8_t i = 0; i < 10; i++)
        {
            HC595_Display(i);
            HAL_Delay(500);
        }
    }
}
```

### 4.2 8 位数码管级联显示
```c
/* 假设 8 片 74HC595 级联 */
void Display_8Digits(uint8_t *digits)
{
    uint8_t i;
    uint8_t buffer[8];
    
    /* 准备段码数据 */
    for(i = 0; i < 8; i++)
    {
        buffer[i] = SEG_CODE[digits[i]];
    }
    
    /* 发送数据 (先发送的在最远端) */
    HC595_SendMultiBytes(buffer, 8);
}

/* 使用示例 */
uint8_t digits[] = {1, 2, 3, 4, 5, 6, 7, 8};
Display_8Digits(digits);
```

### 4.3 动态扫描 (多位数码管)
```c
/* 位选控制数组 (根据实际硬件连接修改) */
const uint8_t DIGIT_SELECT[] = {0xFE, 0xFD, 0xFB, 0xF7, 0xEF, 0xDF, 0xBF, 0x7F};

void Dynamic_Scan(uint8_t *display_data, uint8_t digit_count)
{
    static uint8_t current_digit = 0;
    
    /* 1. 发送段码 */
    HC595_SendByte(SEG_CODE[display_data[current_digit]]);
    
    /* 2. 发送位选 */
    HC595_SendByte(DIGIT_SELECT[current_digit]);
    
    /* 3. 锁存 */
    HC595_Latch();
    
    /* 4. 切换到下一位 */
    current_digit++;
    if(current_digit >= digit_count)
        current_digit = 0;
}

/* 定时器中断中调用 (如 1ms 一次) */
void TIMx_IRQHandler(void)
{
    if(__HAL_TIM_GET_FLAG(&htimx, TIM_FLAG_UPDATE) != RESET)
    {
        __HAL_TIM_CLEAR_FLAG(&htimx, TIM_FLAG_UPDATE);
        Dynamic_Scan(display_buffer, 8);
    }
}
```

## 五、常见问题

### Q1: 显示乱码/全亮/全灭
- 检查段码表是否匹配 (共阴/共阳)
- 检查引脚连接是否正确
- 检查延时是否足够

### Q2: 数码管亮度不足
- 减小限流电阻
- 增加扫描频率
- 检查电源供电

### Q3: 级联时部分不显示
- 检查级联顺序 (先发送的在最远端)
- 检查 /MR 引脚是否接 VCC
- 检查 Q7' 级联连接

### Q4: 共阴 vs 共阳段码表
```c
/* 共阳极 (常用) */
const uint8_t SEG_COMMON_ANODE[] = {
    0xC0, 0xCF, 0xA4, 0xB0, 0x99, 0x92, 0x82, 0xF8,
    0x80, 0x90, 0x88, 0x83, 0xC6, 0xA1, 0x86, 0x8E
};

/* 共阴极 (取反) */
const uint8_t SEG_COMMON_CATHODE[] = {
    0x3F, 0x30, 0x5B, 0x4F, 0x66, 0x6D, 0x7D, 0x07,
    0x7F, 0x6F, 0x77, 0x7C, 0x39, 0x5E, 0x79, 0x71
};
```
