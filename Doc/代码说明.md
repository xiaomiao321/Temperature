# STM32 温度监控系统 - 代码说明

## 1. 项目概述

本项目是一个基于 **STM32F103C8T6** 的 8 路温度监控系统，支持：
- PT100 温度模块数据采集（Modbus RTU 协议）
- 双组独立温度阈值设置
- 超温声光报警
- 继电器联动控制
- 上位机通讯

## 6. 功能模块详解

### 6.1 温度模块 (temperature.c)

**功能**：
- 解析 Modbus RTU 0x04 命令返回的 8 路温度数据
- 超温判断（两组独立阈值）
- 报警锁定/解除
- 继电器和 LED 控制

**数据结构**：
```c
float temperature[8];      // 8 路温度值
uint8_t temp_valid[8];     // 温度有效标志
uint8_t temp_alarm;        // 超温报警标志
uint8_t temp_alarm_latched; // 报警锁定标志
```

**温度数据格式**：
- 16 位有符号整数，放大 10 倍（如 305 = 30.5℃）
- 特殊值 `0xEC78` (-5000) 表示传感器故障

**报警逻辑**：
| 通道 | 比较阈值 | 控制设备 |
|------|----------|----------|
| CH1-CH4 | 阈值 1 | LED、蜂鸣器 |
| CH5-CH8 | 阈值 2 | LED、蜂鸣器 |

### 6.2 温度设置模块 (temp_setting.c)

**功能**：
- 支持两组独立阈值设置（0-999℃）
- 按键操作（SET/加/减/移位/复位）
- 数码管显示控制（正常/设置模式）

**按键操作**：
| 按键 | 短按功能 | 长按功能 |
|------|----------|----------|
| SET | 进入/退出设置 | 切换设置组（1 秒） |
| 移位 | 切换位数（个→十→百） | - |
| 加 | 当前位 +1 | - |
| 减 | 当前位 -1 | - |
| 复位 | 解除报警锁定 | - |

**设置流程**：
1. 按 SET 进入设置模式（当前位闪烁）
2. 长按 SET 1 秒切换组（第一组↔第二组）
3. 按移位选择位数
4. 按加/减修改数值
5. 按 SET 确认并退出

### 6.3 数码管模块 (hc595.c)

**硬件**：两组 74HC595 移位寄存器，每组驱动 3 位共阳数码管

**接口函数**：
```c
void HC595_1_Display3Digits(uint8_t d1, uint8_t d2, uint8_t d3);
void HC595_1_DisplayNumber(uint16_t num);
void HC595_1_Display3DigitsWithBlink(uint8_t d1, uint8_t d2, uint8_t d3, uint8_t blinkPos);
// 第二组数码管类似...
```

**上电自检序列**：
| 显示 | 持续时间 | 说明 |
|------|----------|------|
| 888 | 0.5s | 全段测试 |
| 全灭 | 0.2s | 熄灭测试 |
| 123 | 0.5s | 数字测试 |
| 456 | 0.5s | 数字测试 |
| 000 | 0.3s | 数字测试 |
| 阈值 | 持续 | 正常工作 |

### 6.4 按键模块 (key.c)

**按键配置**：
| ID | GPIO | 功能 |
|----|------|------|
| KEY_ID_SET | PB3 | SET 确认键 |
| KEY_ID_ADD | PB4 | 加键 |
| KEY_ID_SUB | PB0 | 减键 |
| KEY_ID_SHIFT | PB1 | 移位键 |
| KEY_ID_RESET | PA15 | 复位键 |

**MultiButton 配置**：
```c
#define TICKS_INTERVAL    5      // 5ms
#define DEBOUNCE_TICKS    3      // 去抖深度
#define SHORT_TICKS       300    // 短按阈值 300ms
#define LONG_TICKS        1000   // 长按阈值 1000ms
```

### 6.5 继电器控制逻辑

| 状态 | 220V 继电器 (照明灯) | 24V 继电器 (蜂鸣器) | LED |
|------|---------------------|---------------------|-----|
| 正常 | 高电平（开） | 低电平（关） | 灭 |
| 超温 | 低电平（关） | 高电平（开） | 亮 |

## 12. 代码改进建议

### 12.1 架构改进

#### 12.1.1 添加看门狗
**现状**：系统无看门狗保护，死机后无法自恢复
**建议**：启用 IWDG 独立看门狗，在主循环中定期喂狗

```c
// main.c 初始化
IWDG_HandleTypeDef hiwdg;
hiwdg.Instance = IWDG;
hiwdg.Init.Prescaler = IWDG_PRESCALER_64;
hiwdg.Init.Reload = 0xFFF;  // 约 1.5s 超时
HAL_IWDG_Init(&hiwdg);

// 主循环中喂狗
while(1) {
    HAL_IWDG_Reload(&hiwdg);  // 喂狗
    // ... 其他代码
}
```

#### 12.1.2 非易失存储
**现状**：温度阈值断电丢失
**建议**：使用 STM32 内部 Flash 或 EEPROM 保存设置

```c
// 保存阈值到 Flash
void TEMP_SETTING_SaveToFlash(void) {
    HAL_FLASH_Unlock();
    // 擦除并写入 Flash
    HAL_FLASH_Lock();
}

// 上电从 Flash 加载
void TEMP_SETTING_LoadFromFlash(void) {
    // 读取 Flash 中的阈值
}
```

#### 12.1.3 任务调度优化
**现状**：所有任务在主循环顺序执行，无优先级
**建议**：引入简单的时间片调度器

```c
typedef struct {
    void (*task)(void);
    uint32_t interval;  // 执行间隔 (ms)
    uint32_t lastRun;   // 上次执行时间
} Task_t;

Task_t tasks[] = {
    {TEMP_SETTING_Handler, 10, 0},   // 按键处理 10ms
    {TEMP_SETTING_Display, 5, 0},    // 显示刷新 5ms
    {Print_Temperature, 1000, 0},    // 日志输出 1s
};
```

### 12.2 代码质量改进

#### 12.2.1 移除调试变量
**现状**：`temp_setting.c` 中存在大量 `DBG_` 全局变量
**建议**：使用条件编译或完全移除

```c
#ifdef DEBUG
uint16_t DBG_setValue1 = 30;
#endif
```

#### 12.2.2 统一错误处理
**现状**：UART 接收失败无处理
**建议**：添加错误处理和重试机制

```c
if (HAL_UART_Receive(&huart2, &uart2_rx_buf[0], 1, 500) != HAL_OK) {
    // 记录错误，可选择重启串口或复位设备
    Error_Handler_UART2();
    return;
}
```

#### 12.2.3 添加输入验证
**现状**：阈值设置无上下限检查
**建议**：限制阈值范围 (如 0-999℃)

```c
#define TEMP_THRESHOLD_MIN  0
#define TEMP_THRESHOLD_MAX  999

if (*val > TEMP_THRESHOLD_MAX) *val = TEMP_THRESHOLD_MAX;
if (*val < TEMP_THRESHOLD_MIN) *val = TEMP_THRESHOLD_MIN;
```

### 12.3 功能改进

#### 12.3.1 温度校准
**现状**：无温度校准功能
**建议**：添加温度偏移校准

```c
int8_t temp_offset[8] = {0};  // 各通道校准偏移

float Get_Calibrated_Temperature(uint8_t channel) {
    return temperature[channel] + (float)temp_offset[channel] / 10.0f;
}
```

#### 12.3.2 历史数据记录
**现状**：无历史记录
**建议**：添加循环缓冲区记录超温事件

```c
typedef struct {
    uint32_t timestamp;
    uint8_t channel;
    float temperature;
} AlarmEvent_t;

AlarmEvent_t alarm_history[32];  // 记录最近 32 次报警
```

#### 12.3.3 通讯协议增强
**现状**：仅被动接收温度数据
**建议**：支持 Modbus 主站主动查询

```c
// 主动发送查询命令
uint8_t query_cmd[] = {0x01, 0x04, 0x00, 0x00, 0x00, 0x08, 0xF1, 0xCC};
HAL_UART_Transmit(&huart2, query_cmd, 8, 100);
```

### 12.4 性能改进

#### 12.4.1 使用 DMA 接收串口数据
**现状**：使用轮询方式接收 UART2 数据
**建议**：使用 DMA + 空闲中断接收，提高效率

```c
// 初始化 DMA 接收
HAL_UART_Receive_DMA(&huart2, uart2_rx_buf, UART2_BUF_SIZE);

// 空闲中断回调
void HAL_UARTEx_RxEventCallback(UART_HandleTypeDef *huart) {
    if (huart->Instance == USART2) {
        // 处理接收到的数据
    }
}
```

#### 12.4.2 数码管消隐优化
**现状**：闪烁使用固定间隔
**建议**：使用非阻塞定时器实现

```c
// 使用 TIM 定时器产生 500ms 闪烁周期
HAL_TIM_Base_Start_IT(&htim2);  // 500ms 中断
```

#### 12.4.3 减少浮点运算
**现状**：温度使用 float 存储
**建议**：使用定点数 (放大 10 倍存储)

```c
int16_t temperature_x10[8];  // 放大 10 倍存储

// 显示时转换
int temp_int = temperature_x10[i] / 10;
int temp_frac = temperature_x10[i] % 10;
```

### 12.5 安全性改进

#### 12.5.1 继电器状态检测
**现状**：无继电器反馈检测
**建议**：添加继电器状态反馈引脚

#### 12.5.2 传感器故障处理
**现状**：传感器故障仅显示 ERR
**建议**：故障时保持最后有效值或显示 ---

```c
if (temp_raw == TEMP_SENSOR_FAULT) {
    // 显示 --- 表示故障
    HC595_DisplayFault();
}
```

#### 12.5.3 CRC 校验增强
**现状**：使用 Modbus CRC16
**建议**：增加帧完整性验证

```c
uint16_t calc_crc = ModbusCRC16(buf, len - 2);
uint16_t recv_crc = (buf[len-1] << 8) | buf[len-2];
if (calc_crc != recv_crc) {
    return 1;  // CRC 错误
}
```

### 12.6 优先级建议

| 优先级 | 改进项 | 工作量 | 收益 |
|--------|--------|--------|------|
| 🔴 高 | 非易失存储 | 中 | 高 |
| 🔴 高 | 看门狗 | 低 | 高 |
| 🟡 中 | DMA 接收 | 中 | 中 |
| 🟡 中 | 输入验证 | 低 | 中 |
| 🟢 低 | 调试变量清理 | 低 | 低 |
| 🟢 低 | 温度校准 | 中 | 低 |
